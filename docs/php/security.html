  • What is CSRF & how to prevent it in php? 
    •             CSRF stands for Cross-Site Request Forgery, which is a type of attack that occurs when a malicious web site, email, 
            blog, instant message, or program causes a user's web browser to perform an unwanted action on a trusted site 
            for which the user is currently authenticated.

            CSRF attack can be prevented by using a token no which consist of large digit of random number & 
            also put same no. into the session with that form id after form submission check whether session's no 
            & input hidden field values are same or not otherwise it is attacked by CSRF.
  • What is XSS & how to prevent it in php? 
    •             XSS stands for cross site scripting, refers to client-side code injection attack wherein an attacker can 
            execute malicious scripts (also commonly referred to as a malicious payload) into a legitimate website or 
            web application. 

            The simplest and most effective way to prevent XSS attacks is the nuclear option: Ruthlessly escape any character that can affect the structure of your document.

            by using htmlentities() function for users input data we can prevent xss in php.
  • What is Sql Injection 
    •             SQL injections are a method to alter a query in a SQL statement send to the database server.        
            That modified query then might leak information like username/password combinations and can help
            the intruder to further compromise the server.
            
            Sanitize the user data before Storing in database.While displaying the data in browser Convert all 
            applicable characters to HTML entities using htmlentities functions.

            To prevent SQL injections, one should always check & escape all user input, also by using mysql_real_escape_string()
            function() we filter the user input.
